Haga clic en cada fase para desplegar el detalle del proceso

01

Creación de la campaña

El responsable del control inicia la solicitud de información

▾
El responsable accede a la aplicación y crea una nueva campaña. Introduce el nombre de la campaña, el código de control, las fechas de inicio y fin, el responsable principal y alternativo, la fecha límite de respuesta y las reglas de recordatorio automático.
¿Se confirma el inicio de la campaña?
NoLa campaña se guarda como borrador Borrador. No se envía ninguna comunicación.
SíLa campaña queda registrada y se avanza a la siguiente fase.
02

Destinatarios y documentación

Definición de a quién se solicita y qué evidencias son requeridas

▾
El usuario puede definir destinatarios de dos formas: seleccionando primero un manager o socio y asignándole las evidencias necesarias, o bien seleccionando primero las evidencias y asignándolas a varios destinatarios a la vez.
El sistema genera una solicitud individual para cada destinatario con un identificador único. Antes del envío, el usuario puede revisar y ajustar documentación, fechas o personas en copia.
¿Se confirma el envío de las solicitudes?
NoLas solicitudes permanecen editables. No se realiza ninguna acción.
SíEl sistema bloquea la configuración y genera los enlaces individuales para cada destinatario.
03

Envío automático de comunicaciones

El sistema remite un correo individual a cada destinatario

▾
El correo incluye: el código del control, el identificador de la solicitud, la lista de documentación requerida, la fecha límite y el enlace de acceso al entorno de carga.
El estado de cada solicitud cambia a Enviada.
¿El correo se entrega correctamente?
NoEl sistema registra el error. El responsable puede reenviar manualmente.
SíSe queda a la espera de la actuación del destinatario.
04

Acceso y envío de documentación

El destinatario sube archivos y evidencias desde su enlace personal

▾
Si el enlace es inválido o ha caducado, el sistema lo comunica y el proceso queda en pausa hasta que el responsable genere un nuevo acceso.
Si el enlace es válido, el destinatario puede subir archivos, adjuntar evidencias o añadir explicaciones escritas. Admite envíos parciales: no es obligatorio completar todo en un único momento.
Tras cada envío, el sistema notifica automáticamente al responsable. El estado pasa a Parcial o Pendiente de revisión según si la documentación está completa.
05

Revisión por parte del responsable

Validación o rechazo de cada evidencia presentada

▾
El responsable revisa cada documento individualmente: lo marca como válido (queda registrado como aceptado) o lo rechaza indicando el motivo del rechazo.
¿Todas las evidencias obligatorias han sido validadas correctamente?
NoEl sistema genera automáticamente una solicitud de subsanación con únicamente los elementos pendientes. Se envía un nuevo correo al destinatario y el proceso retoma la fase 04.
SíLa solicitud pasa al estado Lista para cerrar.
06

Recordatorios y escalado automático

Notificaciones escalonadas ante la falta de respuesta

▾
El sistema verifica diariamente si existe alguna solicitud pendiente dentro del plazo. Cuando se supera el umbral configurado, activa el siguiente nivel de aviso.
Nivel 1 — Recordatorio enviado únicamente al destinatario principal.
Nivel 2 — Sin respuesta, se incorpora en copia al delegado o asistente del destinatario.
Nivel 3 — Si continúa sin respuesta, el sistema puede incluir en copia a un socio u otro nivel jerárquico superior.
Cada recordatorio queda registrado con fecha y hora. El contador de avisos de la solicitud se incrementa automáticamente.
07

Cierre de la solicitud

El responsable formaliza y bloquea la solicitud completada

▾
Si el responsable decide no cerrar todavía, la solicitud permanece en estado Lista para cerrar sin modificaciones.
¿El responsable confirma el cierre?
Aún noLa solicitud permanece en estado Lista para cerrar.
SíEl sistema bloquea la solicitud y genera automáticamente el informe completo del proceso.
El informe recoge: fecha de envío inicial, fechas de respuesta, número de recordatorios, validaciones, rechazos y fecha de cierre. Estado final: Cerrada.
08

Archivo y reporte final

Almacenamiento estructurado y acceso al panel de campaña

▾
Toda la documentación se archiva en la carpeta correspondiente al código del control y la campaña específica. El informe generado queda almacenado junto con las evidencias.
Desde el panel principal de la campaña, el responsable puede visualizar el estado global, consultar estadísticas de cumplimiento, identificar retrasos y descargar el informe completo para fines de auditoría o control interno.
Escenarios de resolución
Resultado final del proceso
Proceso completado

El destinatario entrega toda la documentación correctamente y dentro del plazo. El proceso finaliza sin recordatorios ni escalados.

Subsanación activada

La documentación es incompleta o incorrecta. El sistema activa automáticamente el circuito de subsanación hasta completar los requisitos.

Sin respuesta

No hay respuesta dentro del plazo establecido. Se activan recordatorios escalonados y el posible escalado jerárquico.

En todos los casos, el sistema garantiza trazabilidad completa, registro cronológico de eventos y archivo estructurado de evidencias.
